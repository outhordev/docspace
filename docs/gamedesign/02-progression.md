## Progression Overview

The game uses a **horizontal progression** model. Players become more versatile, not more powerful. A day-one player and a veteran have the same base stats — the veteran simply has more tools and options available.

### Progression Axes

Player progression branches into four axes:

- **Vessel Upgrades** — Speed, Cargo Capacity, Dive Depth
- **Knowledge / Lore** — Recipes Unlocked, Map Revealed
- **Territory / Base** — Buildings, Defences
- **Reputation / Factions** — Trade Prices, Quest Access

### Vessel Tiers

| Tier | Name | Unlock | Key Feature |
|---|---|---|---|
| 1 | Driftwood Raft | Starting vessel | Slow, small cargo |
| 2 | Sailboat | 50 Driftwood + 10 Coral | Faster, can carry passengers |
| 3 | Ironclad Cutter | Assembly faction quest | Armored, medium cargo |
| 4 | Deep Diver | 20 Echo Fragments | Can explore Undercurrent |
| 5 | Elder Vessel | Endgame quest chain | All abilities, large cargo |

### Reputation System

Each faction tracks player reputation independently on a scale from **-100** (Hostile) to **+100** (Exalted).

| Range | Status | Effects |
|---|---|---|
| -100 to -50 | Hostile | Attacked on sight |
| -49 to -10 | Unfriendly | No trading, limited dialogue |
| -9 to +9 | Neutral | Basic trading |
| +10 to +49 | Friendly | Discounted prices, side quests |
| +50 to +100 | Exalted | Faction-specific vessel, unique recipes |

> [!WARNING]
> Gaining reputation with the Ironclad Assembly will decrease reputation with the Echokeepers, and vice versa. The Tideborn Compact is neutral and unaffected by this tension. Design player choices around this trade-off.

### Anti-Grind Philosophy

- No activity should need to be repeated more than **3 times** to feel like a chore
- Every resource node is unique — different visual, different loot table
- Daily/weekly quests rotate from a pool of **50+** variants
- XP or equivalent does not exist — progression is gated by exploration and crafting, not repetition

> [!DANGER]
> Never add systems that reward time-spent over skill or creativity. This is a core design principle that should influence every feature decision.
